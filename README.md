# Updatecli Helm Charts

[![Release Charts](https://github.com/updatecli/charts/actions/workflows/release.yml/badge.svg)](https://github.com/updatecli/charts/actions/workflows/release.yml)

This repository contains the official [Helm](https://helm.sh/) charts for the [Updatecli](https://www.updatecli.io/) project.

## Charts

| Chart | Description | Version |
|-------|-------------|---------|
| [udash](charts/udash) | Udash — the Updatecli Dashboard web application (backend API + frontend SPA). | [![udash](https://img.shields.io/badge/dynamic/yaml?url=https://updatecli.github.io/charts/index.yaml&query=%24.entries.udash[0].version&label=udash&style=flat-square)](charts/udash/README.md) |
| [udash-agent](charts/udash-agent) | Udash Agent — runs Updatecli on a schedule and ships reports to a Udash instance. | [![udash-agent](https://img.shields.io/badge/dynamic/yaml?url=https://updatecli.github.io/charts/index.yaml&query=%24.entries.udash-agent[0].version&label=udash-agent&style=flat-square)](charts/udash-agent/README.md) |

## Usage

### Add the Helm repository

```console
helm repo add updatecli https://updatecli.github.io/charts
helm repo update
```

### Install a chart

```console
# Udash dashboard
helm install udash updatecli/udash

# Udash agent
helm install udash-agent updatecli/udash-agent
```

See the individual chart READMEs for full configuration options:

- [charts/udash/README.md](charts/udash/README.md)
- [charts/udash-agent/README.md](charts/udash-agent/README.md)

## Development

### Prerequisites

- [Helm](https://helm.sh/docs/intro/install/) 3.x
- [helm-docs](https://github.com/norwoodj/helm-docs) (for regenerating documentation)

### Lint

```console
make lint
```

### Render templates locally

```console
make template
```

### Regenerate documentation

```console
make docs
```

> **Note**: `README.md` files inside each chart directory are auto-generated by `helm-docs` from `values.yaml` annotations and the `README.md.gotmpl` template. Do not edit them directly — edit the template or the annotations instead.
